# First Responder

This doc is more of a collection of ideas and thoughts that I have while working on the First Responder project.

## Raw idea dump

- I waana build a system which could leverage the power of AI to monitor and troubleshoot applications running in a k8s cluster.
- We are gonna build AI agents of-course but less gimmicky.
- The idea is to reduce the involvement of AI agents as much as possible and only involve them when there's an actual need for an intelligence or using it could potentially cover wide range of issues.
- We are gonna build 3 types of agents.
  - Conversational AI Agent: These are the ones which will be served with User interface, i have copilotkit in mind for UI native AI agent experience.
  - Workflow based AI agent: this will run on schedule more like a cron job.
  - Event based AI agent: this will run when an event is triggered.
- All these agents should be able to communicate with each other and share information.
- All should follow certain rules and guidelines to be considered to be a part of the system.
- All need a workspace for them to work on. but i'm yet to decide on the storage system for different type of agents
- ofcourse we need a database to store the metadata of the agents and their workspaces.
- We would need a file system to store the documents and files that are generated by the agents.
- There's a possibility that i will mostly use CAG (Cache Augmented Generative AI) than RAG (Retrieval Augmented Generative AI) for storing the major important information for the agents to work on rather than running the RAG tool call every time.
